# Richlychee OKR (Objectives and Key Results)

## 2026 Q1 OKR

---

## Objective 1: 모든 네이버 스마트스토어 셀러가 가장 먼저 찾는 필수 자동화 도구가 된다

### Why This Matters
개인 셀러부터 대형 사업자까지, 상품 등록의 고통을 완전히 제거하고 본질적인 사업에 집중하게 만드는 것이 우리의 존재 이유입니다.

### Key Results

**KR 1: 사용자 생산성 90% 향상 달성**
- 측정: 평균 상품 등록 시간 수작업 대비 90% 단축 (100개 상품 기준: 10시간 → 1시간)
- 현재: 0% (서비스 미출시)
- 목표: 90%

**KR 2: 구독 전환율 25% 달성**
- 측정: 무료 플랜 사용자 중 유료 플랜으로 전환한 비율
- 현재: 0% (구독 시스템 구현 완료, 사용자 확보 필요)
- 목표: 25%

**KR 3: 사용자 만족도(NPS) 70점 이상 달성**
- 측정: Net Promoter Score (추천 의향 점수)
- 현재: 측정 불가 (출시 전)
- 목표: 70+ (World Class 수준)

### Initiatives (실행 방안)

**생산성 향상을 위한 작업:**
- [ ] 크롤링 성공률 95% 이상 달성 (오류 처리 강화)
- [ ] 이미지 자동 다운로드 및 CDN 업로드 최적화
- [ ] 가격 조정 프리셋 기능 추가 (10%/20%/30% 원클릭)
- [ ] 일괄 카테고리 매핑 기능 구현
- [ ] 엑셀 템플릿 자동 생성 기능

**구독 전환율 향상을 위한 작업:**
- [ ] Free 플랜 사용량 제한 최적화 (너무 제한적이지도, 너무 관대하지도 않게)
- [ ] 결제 프로세스 UX 개선 (3클릭 이내 결제 완료)
- [ ] 플랜별 가치 명확화 (ROI 계산기 제공)
- [ ] 첫 결제 할인 프로모션 (첫 달 50% 할인)
- [ ] 사용량 알림 개선 (80% 도달 시 업그레이드 혜택 강조)

**사용자 만족도 향상을 위한 작업:**
- [ ] 실시간 작업 진행 상황 표시 (진행률 바, 예상 완료 시간)
- [ ] 에러 발생 시 명확한 해결 가이드 제공
- [ ] 고객 지원 채팅 시스템 구축
- [ ] 튜토리얼 비디오 제작 (첫 사용자용)
- [ ] FAQ 및 도움말 페이지 구축

## Objective 2: 국내외 모든 주요 쇼핑몰의 상품 데이터를 99% 정확도로 수집하는 크롤링 엔진을 구축한다

### Why This Matters
단순히 네이버에 등록하는 것을 넘어, 글로벌 이커머스 생태계의 데이터 허브가 되는 것이 장기 비전입니다.

### Key Results

**KR 1: 크롤링 성공률 95% 이상 달성**
- 측정: (성공한 상품 수 / 전체 시도한 상품 수) × 100
- 현재: 측정 불가 (크롤링 시스템 구현 완료, 실제 사용 데이터 필요)
- 목표: 95%

**KR 2: 지원 쇼핑몰 20개 이상 확보**
- 측정: 프리셋으로 등록된 쇼핑몰 수
- 현재: 5개 (쿠팡, 11번가, Amazon, eBay, AliExpress)
- 목표: 20개

**KR 3: 크롤링 속도 평균 100개/분 달성**
- 측정: 동시 크롤링 작업에서 분당 처리 상품 수
- 현재: 측정 불가
- 목표: 100개/분

### Initiatives (실행 방안)

**크롤링 성공률 향상:**
- [ ] 오류 재시도 로직 강화 (3회 재시도, 지수 백오프)
- [ ] Anti-bot 우회 전략 수립 (User-Agent 로테이션, Proxy)
- [ ] 동적 페이지 렌더링 최적화 (Playwright 성능 튜닝)
- [ ] 실패 케이스 자동 분석 및 알림
- [ ] 크롤링 품질 모니터링 대시보드

**지원 쇼핑몰 확대:**
- [ ] 국내: 위메프, 티몬, 인터파크, G마켓, 옥션 추가
- [ ] 해외: Walmart, Target, Rakuten, Tmall, JD.com 추가
- [ ] 각 쇼핑몰별 프리셋 자동 생성 도구 개발
- [ ] 사용자 요청 기반 우선순위 수립
- [ ] 쇼핑몰별 크롤링 가이드 문서화

**크롤링 속도 개선:**
- [ ] 동시성 제어 최적화 (Celery worker 수 조정)
- [ ] 이미지 다운로드 병렬 처리
- [ ] Redis 캐싱 전략 수립
- [ ] 데이터베이스 쿼리 최적화 (N+1 문제 해결)
- [ ] CDN 업로드 배치 처리

## Objective 3: 월 1억 원의 안정적인 MRR(Monthly Recurring Revenue)을 달성하여 지속 가능한 비즈니스 기반을 구축한다

### Why This Matters
기술적 완성도뿐만 아니라, 재무적 지속 가능성이 확보되어야 진정한 프로덕트 마켓 핏을 증명할 수 있습니다.

### Key Results

**KR 1: 유료 구독자 200명 확보**
- 측정: 활성 유료 구독자 수 (Basic + Pro + Enterprise)
- 현재: 0명
- 목표: 200명 (평균 구독료 50만원 기준 = 1억 MRR)

**KR 2: 고객 이탈률(Churn Rate) 5% 이하 유지**
- 측정: 월별 구독 취소 고객 수 / 전체 유료 고객 수
- 현재: 측정 불가
- 목표: 5% 이하 (SaaS 벤치마크: 5~7%)

**KR 3: 고객 생애 가치(LTV) 600만원 이상 달성**
- 측정: LTV = ARPU × (1 / Churn Rate)
- 현재: 측정 불가
- 목표: 600만원 이상 (월 50만원 × 12개월)

### Initiatives (실행 방안)

**유료 구독자 확보:**
- [ ] 랜딩 페이지 제작 및 SEO 최적화
- [ ] 네이버 스마트스토어 커뮤니티 마케팅
- [ ] 유튜브/블로그 튜토리얼 콘텐츠 제작
- [ ] 추천 프로그램 (친구 초대 시 1개월 무료)
- [ ] 온라인 광고 캠페인 (네이버, 구글)

**이탈률 개선:**
- [ ] 구독 취소 시 피드백 수집 시스템
- [ ] 자동 결제 실패 시 재시도 및 알림
- [ ] 사용량 부족 고객 대상 다운그레이드 제안
- [ ] 분기별 고객 만족도 설문조사
- [ ] VIP 고객 전담 지원 채널

**LTV 향상:**
- [ ] 연간 결제 할인 강화 (17% → 25%)
- [ ] 기능 업데이트 정기 공지 (월 1회 뉴스레터)
- [ ] Enterprise 플랜 맞춤 기능 개발
- [ ] 교차 판매 전략 (크롤링 → 자동 등록)
- [ ] 고객 성공 사례 인터뷰 및 공유

## OKR 운영 원칙

### 1. 측정 주기
- **주간 체크인 (매주 월요일)**: Initiatives 진행 상황 확인
- **월간 리뷰 (매월 말일)**: Key Results 달성률 측정 및 조정
- **분기 회고 (분기 마지막 주)**: Objective 달성 여부 평가 및 다음 분기 OKR 수립

### 2. 성공 기준
- Key Results 70% 이상 달성 시 성공으로 간주
- 100% 달성은 목표가 너무 쉬웠다는 신호
- 50% 미만은 목표 재조정 필요

### 3. 투명성
- 모든 OKR은 팀 전체에 공유
- 진행 상황은 실시간 대시보드로 시각화
- 실패도 공유하고 학습

### 4. 우선순위
- 모든 개발 작업은 OKR Initiatives와 연결
- OKR과 무관한 작업은 과감히 제거
- "이 작업이 어떤 KR에 기여하는가?" 항상 질문

## Self-Evaluation (자체 점검)

### ✅ 이 OKR이 잘 작성되었는지 확인

**Good Points:**
1. Objective가 숫자가 아닌 가치 중심으로 작성됨
2. Key Results가 모두 측정 가능한 수치로 정의됨
3. Output(할 일)이 아닌 Outcome(결과)에 집중
4. Initiatives가 구체적이고 실행 가능함
5. 도전적이지만 달성 불가능하지 않은 수준

**Watch Points:**
1. 현재 대부분의 KR이 측정 불가(출시 전) 상태
   → 해결: MVP 출시 후 첫 주간 체크인에서 베이스라인 측정
2. 일부 KR이 너무 공격적일 수 있음
   → 해결: 월간 리뷰에서 실제 데이터 기반으로 조정
